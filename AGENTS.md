# AGENTS.md

## 作業目的

このリポジトリは、JSON形式の集計データからSVGグラフを生成するためのものです。

現在は、以下のSVGを出力します。

- ステータス別件数の横棒グラフ
- グループ別未完了件数の優先度別積み上げ横棒グラフ
- 担当区分別内訳のドーナツチャート

## 基本方針

- まずはPython標準ライブラリのみで動作する構成を維持する。
- 外部ライブラリを追加する場合は、目的・代替案・追加理由をREADMEに記載する。
- SVGは固定サイズで出力し、資料やドキュメントに貼り付けやすい状態を保つ。
- 日本語ラベルはデータ側で管理し、コード内で過度に固定しない。
- 色、フォント、文字サイズ、余白などの見た目は `config/chart_style.jsonc` で管理する。
- SVG内の装飾は最小限にし、単体で見ても意味が分かるように、必要なタイトル・凡例・値ラベルは含める。
- 出力ファイル名は安定させる。定期更新で差し替えやすくするため、原則として毎回同じファイル名で上書きする。

## 変更時の注意

- `data/issue_summary.sample.json` の構造を変える場合は、`docs/component_spec.md` も更新する。
- SVGサイズを変える場合は、出力サイズへの影響を `docs/component_spec.md` に記載する。
- グラフの数値とラベルが一致していることを確認する。
- 凡例、色、ラベルが誤解を招かないことを確認する。
- 優先度のキーと表示名は `data/issue_summary.sample.json` の `priorities` で定義する。
- 優先度の色は `config/chart_style.jsonc` の `colors.priority` で定義する。
- 完了・アーカイブなど管理完了系は、緑またはグレー系を基本とする。

## 完了条件

- `python scripts/generate_svg.py --input data/issue_summary.sample.json --output output` がエラーなく実行できる。
- `output/` 配下にSVGが生成される。
- 生成されたSVGに文字化け・大幅な崩れがない。
- READMEの実行手順が最新である。
